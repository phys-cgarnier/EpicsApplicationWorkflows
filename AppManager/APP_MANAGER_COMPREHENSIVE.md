# IOC Manager Application - Comprehensive Design Document

## Executive Summary
This document defines the end-to-end workflows and comprehensive design for the SLAC Cryoplant IOC Manager application. The design ensures all IOC configuration files are properly managed, validated, formatted, and synchronized across the entire system.

## Core Principles
1. **Safety First**: All changes require preview and approval, with automatic backups
2. **Consistency**: Enforce standards across all IOC configurations
3. **Traceability**: Track all changes with audit logs and version history
4. **Automation**: Reduce manual work while maintaining control
5. **Validation**: Catch errors before they reach production

## 1. WORKFLOW DEFINITIONS

### 1.1 Substitution File Management Workflow

#### User Story
"As an IOC engineer, I need to clean up and standardize substitution files while ensuring consistency and preventing errors"

#### End-to-End Process:

**Step 1: Discovery & Analysis**
- User selects IOC system from left panel
- System scans all substitution files for that IOC
- Dashboard displays:
  - File count and total size
  - Last modification dates
  - Quick validation status (green/yellow/red indicator)

**Step 2: Formatting & Validation**
- User clicks "Format Substitutions" button
- System performs multi-stage analysis:
  ```
  Stage 1: Syntax Check
  - Validate brace matching (detect ")" instead of "}")
  - Check column count consistency
  - Verify macro syntax $(MACRO_NAME)

  Stage 2: Quote Consistency Analysis
  - Identify quoted vs unquoted values
  - Apply rules:
    * Strings with spaces → require quotes
    * Numeric values → no quotes
    * Macros → no quotes
    * Descriptions → always quoted
    * Empty values → quoted as ""

  Stage 3: Column Alignment
  - Calculate optimal column widths
  - Preserve readability with proper spacing
  - Maintain comment positions

  Stage 4: Macro Resolution
  - Verify all macros are defined
  - Check for unused macros
  - Validate macro value ranges
  ```

**Step 3: Issue Detection & Correction**
- System presents findings in categorized panel:
  ```
  CRITICAL (Requires Fix):
  ✗ Line 45: Mismatched brace - ")" should be "}"
  ✗ Line 67: Column count mismatch (expected 7, found 6)
  ✗ Line 89: Undefined macro $(UNKNOWN_MACRO)

  WARNINGS (Recommended):
  ⚠ Line 23: Inconsistent quoting - "1234" should be 1234
  ⚠ Line 34: Description missing quotes
  ⚠ Line 56: Duplicate entry detected

  SUGGESTIONS (Optional):
  ℹ Line 12: Consider using macro for repeated value "CP15"
  ℹ Multiple: 15 values could use standard macros
  ```

**Step 4: Preview & Approval**
- Side-by-side diff view showing:
  - Original file (left)
  - Proposed changes (right)
  - Color-coded changes (additions/deletions/modifications)
- User can:
  - Accept all changes
  - Accept selected changes
  - Modify individual corrections
  - Add comments for audit trail

**Step 5: Backup & Apply**
- System creates timestamped backup:
  ```
  Original: 2kcb_AIs.substitutions
  Backup:   backups/2024-12-19/2kcb_AIs.substitutions.20241219_143022.backup
  ```
- Apply approved changes
- Update validation status
- Log changes to audit system

**Step 6: Verification**
- Run post-format validation
- Generate report:
  ```
  Formatting Complete:
  - Files processed: 12
  - Issues fixed: 34
  - Warnings resolved: 15
  - Backup location: /backups/2024-12-19/
  - Validation status: PASSED
  ```

### 1.2 Archive File Synchronization Workflow

#### User Story
"As an operations engineer, I need to ensure all PVs are properly archived with appropriate sampling rates"

#### End-to-End Process:

**Step 1: PV Extraction**
- Parse all substitution files to extract PVs
- Build complete PV inventory with metadata:
  ```
  PV Name: CFT:CP15:1011:FLOW
  Source: 2kcb_AIs.substitutions
  Template: cft_flow.vdb
  Type: Analog Input
  Expected Range: 0-100 g/s
  Update Rate: 1 Hz
  ```

**Step 2: Archive Analysis**
- Compare PV inventory against archive files
- Generate coverage report:
  ```
  System: 2kcb
  Total PVs: 1,247
  Archived: 1,189 (95.4%)
  Missing: 58 (4.6%)
  Orphaned: 23 (archived but no longer exist)
  ```

**Step 3: Smart Archive Generation**
- Apply intelligent rules based on PV type:
  ```
  Rules Engine:
  - Temperature sensors → 10 second sampling
  - Pressure sensors → 5 second sampling
  - Valve states → on change
  - Alarms → on change
  - Flow meters → 1 second sampling
  - Calculated values → monitor mode
  ```
- Generate proposed archive entries:
  ```
  # Generated by IOC Manager - 2024-12-19
  # Missing PVs for 2kcb system

  # Pressure sensors - 5 second sampling
  CPT:CP15:2001:PRESS 5 scan
  CPT:CP15:2002:PRESS 5 scan

  # Valve states - on change
  CVL:CP15:3001:STATE 1 monitor
  CVL:CP15:3002:STATE 1 monitor
  ```

**Step 4: Review & Optimize**
- Display proposed changes with justification
- Allow rate adjustments:
  - Show impact on storage
  - Estimate network bandwidth
  - Suggest optimizations
- User can override defaults

**Step 5: Integration & Deployment**
- Update archive configuration files
- Validate syntax
- Generate deployment script:
  ```bash
  # Archive configuration update script
  # Generated: 2024-12-19 14:35:22

  # Backup existing configuration
  cp /archive/2kcb.archive /archive/backup/2kcb.archive.20241219

  # Apply new configuration
  cp /tmp/2kcb.archive.new /archive/2kcb.archive

  # Restart archiver (requires confirmation)
  systemctl restart archiver-2kcb
  ```

### 1.4 Backup Management Workflow

#### Automated Backup Strategy:

**Directory Structure:**
```
backups/
├── 2024/
│   ├── 12/
│   │   ├── 19/
│   │   │   ├── 2kcb_AIs.substitutions.20241219_143022.backup
│   │   │   ├── 2kcb_AIs.substitutions.20241219_151545.backup
│   │   │   └── manifest.json
│   │   └── 18/
│   └── 11/
└── retention.policy
```

**Retention Policy:**
```json
{
  "daily_backups": {
    "keep_days": 7,
    "max_versions_per_file": 10
  },
  "weekly_backups": {
    "keep_weeks": 4,
    "archive_format": "tar.gz"
  },
  "monthly_backups": {
    "keep_months": 12,
    "archive_format": "tar.gz",
    "compression": "maximum"
  },
  "permanent_backups": {
    "trigger": "major_version",
    "location": "archive/permanent/"
  }
}
```

**Backup Operations:**
- Automatic backup before any modification
- Intelligent deduplication (only backup if changed)
- Compressed storage for old backups
- Quick restore capability with preview
- Backup pruning based on policy

## 5. KEY DESIGN DECISIONS

### Quote Handling Rules
```python
QUOTE_RULES = {
    'numeric': NO_QUOTES,          # 123, 45.6, -7.8
    'boolean': NO_QUOTES,          # true, false, TRUE, FALSE
    'macro': NO_QUOTES,            # $(MACRO_NAME)
    'string_no_space': OPTIONAL,   # abc, test123
    'string_with_space': REQUIRED, # "hello world", "test value"
    'empty': QUOTE_REQUIRED,       # ""
    'description': ALWAYS_QUOTED,  # "This is a description"
    'path': QUOTE_IF_SPACES,      # /path/to/file or "/path with spaces/file"
}
```
